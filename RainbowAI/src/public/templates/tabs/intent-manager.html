4-Tier Intent Classification System

Rainbow uses a cascading 4-tier system that progresses from fastest to smartest. Each tier is checked in order until a match is found. This ensures optimal speed while maintaining accuracy.

Tier Speed Accuracy Best For Example
üö® Priority Keywords ~0.1ms ‚ö†Ô∏è Strict Critical emergencies (fire, theft, ambulance) /fire|kebakaran/i
‚ö° Smart Matching ~1ms 88-92% Direct keywords, abbreviations, typos "wifi", "wfi", "WiFi" all match
üìö Learning Examples ~50ms 92-95% Natural variations, paraphrases "how much" = "what's the cost"
ü§ñ AI Fallback 100-500ms 90-93% Complex queries, edge cases, fallback Full AI classification

üí° Pro Tip: The Cascading Flow

95%+ queries should match in T1-T3 (under 50ms)! If your T4 usage is high (>20%), add more keywords/examples to T2/T3. Monitor tier usage in üìä Monitor ‚Üí Performance.

When to use: Critical situations requiring instant detection

Quick Steps:

  1. 1. Click the chevron on T1 section
  2. 2. Enter pattern: /fire|kebakaran/i
  3. 3. Add description: "Fire emergency"
  4. 4. Click Add ‚Üí Save Patterns
  5. 5. ‚ö†Ô∏è Restart server to apply

Troubleshooting Common Issues

‚ùå Keywords not matching

Possible causes:

  • ‚Ä¢ Keyword in wrong intent - check with Test Console
  • ‚Ä¢ Fuzzy threshold too high - lower from 0.80 to 0.75 in T4 settings
  • ‚Ä¢ Forgot to save - click "Save Keywords" button
  • ‚Ä¢ Cache issue - reload page with Ctrl+Shift+R

‚ùå High T4 usage (>20%)

Solutions:

  • ‚Ä¢ Check Performance tab to see which queries hit T4
  • ‚Ä¢ Add those queries as T2 keywords or T3 examples
  • ‚Ä¢ Lower confidence thresholds (T2: 0.75, T3: 0.65)
  • ‚Ä¢ Add more training examples for common patterns

‚ùå Wrong intent detected

Debug steps:

  • ‚Ä¢ Use Test Console to see which tier/keyword matched
  • ‚Ä¢ Check for duplicate keywords across intents
  • ‚Ä¢ Verify regex patterns (T1) aren't too broad
  • ‚Ä¢ Review semantic examples for conflicts
